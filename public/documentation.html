Documentation page for my Movie App

This project is about building the server-side component of a “movies” web application.

The web application will provide users with access to the following information:

- movies
- directors and
- genres.

Users will be able to sign up, update their personal information, and create a list of their favorite movies.

List of URL endpoints, defining Request, URL and Response.

Returns data about a a user by name. It also returns information about an updated user by name or a user that was deleted by username.
S/N REQUEST URL RESPONSE
1 GET /movies A JSON object holding data about all movies. Returns a list of all movies to the user.
2 GET /movies/:Title A JSON object holding data about a movie containing. Returns data about a movie by title.
3 GET /genre/:Name A JSON object holding data about a movie with genre and director. Returns data about a movie with genre information by name.
4 GET /director/:Name A JSON object holding data about a movie with director information. Returns data about a movie with director information by name.
5 GET /documentation A HTML Page holding information about myFlixDB App. Returns documentations about myFlixDb App.
6 GET /users/:Username A JSON data holding information about a user.
7 GET /users/:Username/movies/:MovieI A JSON data holding information about a movie that was added to a user's list of favorites. It also used to display movies thatb was deleted from user's list of favorites.
8 GET /movies/:Title A JSON object holding data about a movie containing title. Returns data about a movie and title.
9 GET /genre/:Name/:director/:Name/imagepath A JSON object holding data about a movie genre, direcor by name and the imagepath. Returns a mnovie showing the genre, director name and ImagePath.
10 GET /movies/:Title/director/:Name/:Bio/:Birth//:death A JSON object holding data about a movie containing title director name, Bio including birth and death information.
11 POST /users A text message showing name, password, email, birthdate and favourite movies of user that was registered.
12 PUT /users A text message showing the user details that was updated. Allow new user to register.
13 PUT /users/:Username A text message showing the user details that was updated. Allow users to update their information.
14 DELETE /users/[id] A text message indicating if the user was removed successfully.
